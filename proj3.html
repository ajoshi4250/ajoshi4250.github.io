#Part 2: Code (K-Nearest Neighbors)
xtrain, xtest, ytrain, ytest = train_test_split(wine.data, wine.target, stratify=wine.target, random_state=20)

trainacc = []
testacc = []
scoresa = []
scoresf = []
error = []

nnum = range(1,21)

for i in nnum:
    knn = KNeighborsClassifier(n_neighbors=i)
    knn.fit(xtrain,ytrain)
    trainacc.append(knn.score(xtrain, ytrain))
    testacc.append(knn.score(xtest, ytest))
    a = cross_val_score(estimator=knn, X=wine.data, y=wine.target, cv=10, scoring='accuracy')
    f = cross_val_score(estimator=knn, X=wine.data, y=wine.target, cv=10, scoring='f1_micro')
    e = cross_val_score(estimator=knn, X=wine.data, y=wine.target, cv=10, scoring='neg_root_mean_squared_error')
    scoresa.append((i, a.mean()))
    scoresf.append((i, f.mean()))
    error.append((i, -1*e.mean()))

mp.plot(nnum, trainacc, label='Training Set Accuracy')

Part 2: Explanation

In this part, I use the wine dataset provided by SciKit Learn, originally published by UCI. This data looks at wine made by three different cultivars who used grapes from the same region of Italy. This dataset attempts to predict which cultivar made the wine. It does this by looking at 13 different numeric attributes of the wine. Below the section titled, "#Part 2: Code (initialization)," a full length description of these attributes can be found. The two additional algorithms I have chosen are K Nearest Neighbors (KNN) and Decision Trees. The hyperparameter for KNN is the number of neighbors and for decision trees, it's the max depth of the tree. For both of these, I did an exhaustive search method to find the optimal value. I chose a reasonable range of values and plotted their accuracy and then chose the value that had the highest accuracy with the testing set. The graphs are below the sections for each algorithm. I have also incuded the confusion matrix for the hyperparameter values that I chose for both of the algorithms. Looking at this and the graphs of their accuracies over many values of the hyperparameter, it is clear that the Decision Trees algorithm is outperforming the KNN algorithm, especially since the best accuracy that KNN has is just above 0.7 with K=1 compared to Decision Tree's best of 0.9 with a max depth of 5. While one can see that while a max depth of 5 is performing better for Decision Trees for the split I selected, a max depth of 3 is actually doing better using 10 fold cross validation. For KNN, the cross validation supports the idea that K=1 has the best performance overall. Under the cross validation scores for accuracy, I also provide the cross validation scores for f1 and the RMSE.
